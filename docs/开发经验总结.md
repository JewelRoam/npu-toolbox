# NPU工具箱开发经验总结

## 项目概述

NPU工具箱是一个基于 Tauri v2 + React + TypeScript 的桌面应用程序，旨在让用户能够轻松使用本地 AI 能力，特别是利用 NPU（神经处理单元）进行加速。

## 技术栈选择

### 前端
- **React 18**: 使用函数组件和 Hooks
- **TypeScript**: 类型安全
- **Tailwind CSS**: 快速样式开发
- **React Router**: 页面路由
- **Vite**: 快速开发构建

### 后端
- **Tauri v2**: Rust 桌面应用框架
- **tokio**: 异步运行时
- **serde**: JSON 序列化/反序列化
- **log/env_logger**: 日志系统

## 架构设计

### 前后端通信

使用 Tauri 的 `invoke` API 进行前后端通信：

```typescript
// 前端调用
import { invoke } from '@tauri-apps/api/core'
const hardwareInfo = await invoke<HardwareInfo>('get_hardware_info')

// 后端处理
#[tauri::command]
pub async fn get_hardware_info() -> Result<HardwareInfo, String> {
    // ...
}
```

### 数据流

```
前端 UI → invoke() → Rust Command → PowerShell/系统API → 返回数据 → 前端渲染
```

## 开发过程中的挑战与解决方案

### 1. Tauri v2 配置变化

**问题**: Tauri v2 与 v1 配置格式差异较大，很多插件和 API 发生了变化。

**解决方案**: 
- 使用 `tauri-plugin-shell` 替代旧的 shell API
- 在 `tauri.conf.json` 中正确配置 `app.security.csp`
- 使用 `generate_context!()` 宏

### 2. Windows 构建环境

**问题**: Windows 下需要 MinGW-w64 工具链来编译 Rust 代码。

**解决方案**:
```bash
# 安装 MSYS2 后安装 MinGW
pacman -S mingw-w64-x86_64-gcc

# 配置 PATH
$env:PATH = "C:\msys64\mingw64\bin;" + $env:PATH
```

### 3. 硬件信息获取

**问题**: Rust 没有直接的 Windows 硬件信息 API。

**解决方案**: 通过 PowerShell 命令获取硬件信息：
```rust
fn run_powershell(script: &str) -> Result<String, String> {
    let output = Command::new("powershell")
        .args(["-NoProfile", "-Command", script])
        .output()
        .map_err(|e| e.to_string())?;
    Ok(String::from_utf8_lossy(&output.stdout).to_string())
}
```

### 4. 异步处理

**问题**: 硬件信息获取需要并行处理以提高效率。

**解决方案**: 使用 `tokio::join!` 并行执行：
```rust
let (cpu, gpu, memory, storage, npu) = tokio::join!(
    get_cpu_info(),
    get_gpu_info(),
    get_memory_info(),
    get_storage_list(),
    detect_npu_internal()
);
```

### 5. 类型安全

**问题**: 前后端数据类型需要保持一致。

**解决方案**: 定义共享的接口/结构体：
```typescript
// 前端
interface HardwareInfo {
  cpu: CPUInfo;
  gpu: GPUInfo;
  // ...
}
```

```rust
// 后端
#[derive(Debug, Serialize, Deserialize, Clone)]
pub struct HardwareInfo {
    pub cpu: CPUInfo,
    pub gpu: GPUInfo,
    // ...
}
```

## 最佳实践

### 1. 错误处理

- 使用 `Result<T, String>` 作为命令返回类型
- 提供默认值/模拟数据作为 fallback
- 前端显示友好的错误提示

### 2. 日志规范

- 使用 `log` crate 进行日志记录
- 日志信息使用英文，便于国际化
- 合理使用 `info!`, `warn!`, `error!` 级别

### 3. 代码组织

```
src-tauri/
├── src/
│   ├── main.rs      # 应用入口
│   └── commands.rs  # Tauri 命令
└── Cargo.toml

src/
├── components/      # 可复用组件
├── pages/           # 页面组件
├── App.tsx          # 根组件
└── main.tsx         # 入口文件
```

### 4. 设置持久化

使用用户配置目录存储设置：
```rust
let config_dir = dirs::config_dir()
    .ok_or("Failed to get config directory")?
    .join("NPUToolbox");
```

## 性能优化建议

1. **前端**: 使用 React.memo 减少不必要的重渲染
2. **后端**: 使用异步并行获取硬件信息
3. **缓存**: 对频繁请求的数据实现缓存机制
4. **懒加载**: 对大型组件使用 React.lazy
